## Issues Found and Status

### 1. Broker mismatch — FIXED ✅
**Problem:** Wokwi firmware binary was stale — built with old broker config. Python and ESP32 were on different brokers.  
**Fix:** Rebuilt firmware with `pio run`. Both now use `broker.emqx.io:1883`.  
**Files:** `firmware/wokwi/esp32-home-node/config.h`, `.env`

### 2. MQTT response timeout too short — FIXED ✅
**Problem:** Python waited 1 second for ESP32 response. Wokwi runs at 35% simulation speed, causing consistent timeouts.  
**Fix:** Increased timeout from `1.0` to `10.0` seconds.  
**File:** `iot_controller.py:113`

### 3. Encoding bugs in agent.py — FIXED ✅
**Problem:** Two strings had UTF-8/latin-1 mojibake corruption (`Ã©` instead of `é`).  
**Fix:** Corrected both strings.  
**File:** `agent.py` lines 452, 640

### 4. Gas alert not delivered — FIXED ✅
**Problem:** Gas alert fired on MQTT topic `robocompagnon/home/alerts/gas` but no delivery mechanism outside the dashboard.  
**Fix:** Added `_GasAlertTelegramNotifier` in `iot_controller.py`. Subscribes to gas alert topic, sends Telegram message if `TELEGRAM_BOT_TOKEN` and `TELEGRAM_CHAT_ID` are set in `.env`.  
**Files:** `iot_controller.py`

### 5. No single entry point — FIXED ✅
**Problem:** Two Streamlit apps exist (`app.py` = old robot UI, `app_complet.py` = IoT dashboard). Confusing.  
**Fix:** Added `run.py` launcher that starts `app_complet.py`. Use `py run.py` to start.  
**File:** `run.py` (new)

### 6. Wokwi Terminal blank (no serial output) — OPEN ⚠️
**Problem:** Wokwi VS Code Community License shows no serial output in terminal. Root cause unclear — may be ELF/binary mismatch timing or Community License limitation.  
**Workaround:** Test by sending a command and checking if LED changes in Wokwi diagram.  
**Impact:** Debugging only — simulation may still function correctly.

### 7. Telegram bot commands not wired — OPEN ⚠️
**Problem:** `telegram_bot.py` can send notifications but cannot receive commands from Telegram (no polling loop, no command handler).  
**Impact:** One-way only — alerts go out, no control from Telegram.  
**Effort:** Medium — needs polling loop + command routing to `iot_controller`.

### 8. Only one room — KNOWN LIMITATION ℹ️
System locked to `living_room`. Parser defaults to it. No multi-room support.  
Not a bug, just a scope boundary.

### 9. Occupancy sensor is a slide switch — KNOWN LIMITATION ℹ️
Wokwi `diagram.json` uses a slide switch for PIR occupancy sensor. Fine for simulation. Edge logic using occupancy uses this simulated value.
